AT&T Drive UI Kit

This AT&T Drive Automotive App development UI Kit is built to enable you -the developer, to effortlessly create apps to be deployed on AT&T Drive Automotive Services Delivery Platform.

Building apps for Head Unit

App framework contains all the essentials to build standardized head unit apps, and starting your app from the sample apps is the fastest way to get started. Then take advantage of this site to visualize the UX, and get the example code. Now you are ready to build your first app, and the API is readily available.

Seed your first app

After installing the AT&T Drive Head Unit SDK locally, using terminal in "Connected-Car-SDK" folder run this command: grunt seed --dec-host=127.0.0.1 --dec-port=4402 -name=/full/path/to/your/app (you can change the default IP and port if needed).
You can always change ports and IP later (see optional commands in Setup.MD).

Once the build is completed the app will load in your browser and if you see the first page, the app is connected to span and ready for development.

You can also start developing from the existing sample app.

Voice as an app concept

AT&T Drive is a cloud based automotive services delivery platform and the apps have the ability to be fully voice enabled as part of AT&T initiative to prevent distracted driving. This document, including the design guidelines and the Information Architecture stencils are simplified to enable creation of automotive apps that are effortless and distraction-free.

Voice as an app flow:

Common source code, less clutter, faster apps, better experience

The intent of this toolkit is to help automotive manufacturers maintain their brand presence in the in-car digital experience. As app developers build apps from the same framework of libraries and APIs, the result is structured, seamless, easy brand-able set of applications, and as AT&T Drive offers theming per vehicle make/model/trim, reusing common UI libraries and app source-code is crucial.
